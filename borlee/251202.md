예전에 공부했던 코드를 백트래킹으로 바꿔보려고 펼쳤는데..
타입힌트를 추가하고 코드를 개선하다가 예상한 시간을 초과해버렸다. 
여기서 사용했던 combination_with_replacement(iterable, r)은 중복을 허용하고 순서는 고려하지 않는 조합을 만들 떄 사용할 수 있는 함수다.



**코드(선택)**
```python
from itertools import combinations_with_replacement
from collections import Counter as CounterClass
from typing import List, Tuple, Dict, Counter


def solution(n: int, info: List[int]) -> List[int]:
    maxdiff: int = 0
    max_comb: Dict[int, int] = {}
    
    def calculate_score(cnt: Counter[int]) -> Tuple[int, int]:
        ryon: int = 0
        apeach: int = 0
        
        for score in range(1, 11): 
            ryon_arrows: int = cnt.get(score, 0)
            apeach_arrows: int = info[10 - score]  
            
            if ryon_arrows > apeach_arrows:
                ryon += score
            elif apeach_arrows > 0:
                apeach += score
        
        return ryon, apeach

    for combi in combinations_with_replacement(range(11), n):
        cnt: CounterClass = CounterClass(combi)
        score_ryon, score_apeach = calculate_score(cnt)
        diff: int = score_ryon - score_apeach

        if diff > maxdiff:
            maxdiff = diff
            max_comb = dict(cnt)
    
    if maxdiff > 0:
        answer: List[int] = [0] * 11
        for target, arrows in max_comb.items():
            answer[10 - target] = arrows
        return answer
    
    return [-1]

```
