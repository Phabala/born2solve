우선순위큐는 만날 때마다 어렵..
구조체에서 r = row, c = column에 해당한다.  std:: priority_queue는 기본적으로 최대힙으로 동작하므로 < 연산 결과를 오버로딩해서 최소힙으로 만들었다.
람다 식을 사용해도 된..........다..고(C++11 이후)
```c++
auto cmp = [](const Pos& p1, const Pos&p2) {
	return p1.heightDiff > p2.heightDiff;
};

std::priority_queue<Pos, std::vector<Pos>, decltype(cmp) > pq(cmp);
```

deltype = declared type

아래 코드는 그리디(중에서도 프림) + 우선순위큐를 활용한 코드다.(공부용)
visited 시점을 push가 아니라 pop할 때 하는 것도 조금 골아프다.
( 이 이유는 큐에 넣을 때 체크하면 나중에 발견된 더 짧은 경로가 무시될 수 있기 때문이다.)
isValid함수는 경계와 상태를 검사하는 것인데 특히 !visited[nr][nr] 은 중복 방문으로 인한 무한 루프나 불필요한 연산을 차단하는 데 중요하다. 그냥 늘여놓은 것처럼 보이지만 연산자 순위도 고려하고 있다.
사다리 비용 발생과 사다리 비용 없이 이동하는 것도 너무 센스있게 짠 것 같다. 
효율적인 dx, dy는 암기가 답이다.... 북->동->남->서

**코드(선택)**
```c++
#include <string>
#include <vector>
#include <algorithm>
#include <queue>

using namespace std;

struct Pos {
    int r;
    int c;
    int heightDiff;
    bool operator<(const Pos &p) const { return heightDiff > p.heightDiff; } 
};

int dy[4] = {-1, 0, 1, 0}, dx[4] = {0, 1, 0, -1};
bool visited[301][301];

bool isValid(int nr, int nc, int rows, int cols) {
    return nr >= 0 && nr < rows && nc >= 0 && nc < cols && !visited[nr][nc];
}

int solution(vector<vector<int>> land, int height) {
    int totalCost = 0;
    int rows = land.size(), cols = land[0].size();
    
    priority_queue<Pos> pq; 
    pq.push({0, 0, 0});
    
    while(!pq.empty()) {
        Pos current = pq.top();
        pq.pop();
        
        if (visited[current.r][current.c])
            continue;
        totalCost += current.heightDiff;
        for (int i = 0; i < 4; i++) {
            int nr = current.r + dy[i], nc = current.c + dx[i];
            if(isValid(nr, nc, rows, cols)) {
                int diff = abs(land[current.r][current.c] - land[nr][nc]);
                if (diff > height)
                    pq.push({nr, nc, diff});
                else
                    pq.push({nr, nc, 0});
            }
        }
        visited[current.r][current.c] = true;
    }
    return totalCost;
    
}
```
